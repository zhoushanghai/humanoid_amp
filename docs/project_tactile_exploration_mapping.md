# 主动探索触觉建图任务说明

- Date: 2026-03-10
- Type: Project Description

## 项目目标

基于当前 G1 AMP 训练仓库，新增一个“黑暗环境下的主动探索建图”任务：机器人通过主动扫掠与接触，从本体感知中提取“自由空间证据 + 接触占据证据”，逐步构建局部 3D 占据图，并利用学习到的结构先验做局部补全。

## 项目背景

- 当前仓库已经具备 AMP 运动控制基础能力，现有任务注册位于 `__init__.py`，训练入口位于 `train.py`，环境配置与奖励逻辑分别集中在 `g1_amp_env_cfg.py` 与 `g1_amp_env.py`，适合继续扩展为“运控 + 感知”联合任务。
- 相比纯速度跟踪或纯风格模仿，本任务直接面向“无视觉或弱视觉条件下的局部建图”，更接近真实探索场景。
- 本任务的核心价值不只是摸到障碍物，而是把接触和未接触的扫掠轨迹都转化为可学习的空间证据。

## 系统范围

- 阶段 1：训练面向主动探索的 AMP 运动控制策略。
- 阶段 2：基于阶段 1 策略在 Isaac Lab 中采集数据集，离线训练本体感知到局部 3D 体素占据图的感知模型。
- 阶段 3：联合调试运控与感知，使探索策略更精准、更有利于高效建图。

## 当前不包含

- 第一阶段暂不考虑下蹲动作。
- 脚步动作只覆盖跨越很矮障碍物，不做复杂越障或攀爬。
- 第一版只做局部占据建图，不追求全局 SLAM。
- 第一版不直接承诺真机部署。

## 系统构成

### 模块 A：主动探索运控策略

- 目标：在保持 AMP 风格约束的前提下，让机器人主动用手探索圆柱体、立方体、墙面和墙角。
- 行为重点：
  - 手部扫掠与触摸动作为主。
  - 不要求下蹲。
  - 足部只需要支持跨越较矮障碍物，保证接近与 reposition。

### 模块 B：本体感知到局部占据图的感知模型

- 输入：过去 `0.5s ~ 1.0s` 的本体感知时间序列。
- 输出：局部 `3D voxel occupancy`。
- 能力目标：
  - 把“摸到哪里”转成接触占据证据。
  - 把“手臂/身体扫过但未接触哪里”转成自由空间证据。
  - 借助结构先验补全未直接观测到的局部几何。

### 模块 C：联合策略层

- 目标：让控制策略和感知模型互相增益，而不是各自独立最优。
- 输出形态待比较：
  - 显式高层指令。
  - latent 条件。
  - 端到端感知特征注入。

## 技术路线一：AMP 运动控制模型

### 目标

训练一个可以稳定主动触摸目标结构、并保持 AMP 风格先验的探索策略。

### 训练数据

- 使用黑暗探索数据集作为 AMP 动作先验来源。
- 数据重点应覆盖：
  - 摸圆柱体
  - 摸立方体
  - 摸平整墙面
  - 摸墙角/拐角
- 数据暂不要求下蹲轨迹。

### 奖励设计

- `style reward`
  - 维持动作自然性与 AMP 风格一致性。
- `exploration reward`
  - `contact_count_reward`
    - 鼓励产生有效接触点。
  - `novel_surface_grid_reward`
    - 将障碍物表面离散成网格。
    - 每触摸到一个新格子，给予一次性新颖性奖励。
  - `geometry_weighted_novelty_reward`
    - 普通表面：`1`
    - 棱边 / 圆柱边缘：`5`
    - 尖角 / 拐点：`10`
  - `free_space_sweep_reward`（建议加入）
    - 当手部或前臂在未碰撞前扫过一段空间时，记录为自由空间证据。
    - 该项不是“摸到”奖励，而是“有效扫描”奖励，用于对接后续建图。

### 阶段产出

- 可复用的探索策略 checkpoint。
- 与触摸覆盖率相关的训练曲线。
- 触摸轨迹与接触标签，为阶段 2 采集脚本提供行为先验。

## 技术路线二：数据集采集与离线感知训练

### 目标

利用阶段 1 的探索策略，在 Isaac Lab 中批量采集“本体感知时间序列 -> 局部 3D 占据图”的监督数据，训练离线感知模型。

### 输入

- `0.5s ~ 1.0s` 的 proprioceptive history
- 以 `50Hz` 为例，对应过去 `25 ~ 50` 帧
- 每帧建议包含：
  - 关节位置偏移量 `joint position error`
  - 关节速度 `joint velocities`
  - 关节输出力矩或动作 `joint torques / actions`
  - IMU 数据 `base angular velocity`
  - IMU 数据 `projected gravity`
  - action

### 输出

- 局部 `3D voxel occupancy`

### 候选模型

- `Transformer`
- `RNN`
- `Diffusion-style policy / generative decoder`

说明：
- 当前只确定采用 Encoder-Decoder 主架构。
- 你提到的部分更细模型内容“暂时无法在飞书文档外展示”，因此这里先保留为候选路线，不展开未确认细节。

### 数据集字段

- 历史本体感知序列
- 接触事件与接触部位
- 采集时的局部体素真值
- 可选：自由空间扫掠掩码
- 可选：目标物体类别与局部几何属性标签（平面、边缘、角点）

### 阶段产出

- 离线训练好的感知模型。
- 局部体素重建结果与可视化样例。
- 不同序列模型的横向比较结果。

## 技术路线三：联合训练与调试

### 目标

在已有运控模型和感知模型基础上，提升探索效率、接触有效性与建图质量。

### 方案 1：冻结底层，训练第三个高层规划模型

- 冻结阶段 1 的运控模型。
- 冻结阶段 2 的感知模型。
- 训练一个第三个高层策略，输出：
  - 显式探索指令，或
  - latent 控制信号。

优点：
- 训练更稳定。
- 更容易定位问题来自高层规划还是底层控制。

风险：
- 受到底层模块表达能力上限约束。

### 方案 2：解冻运控与感知，端到端联合训练

- 将感知模型输出直接接入运控模型输入。
- 共同优化探索行为与建图效果。

优点：
- 理论上能得到更强的任务协同。

风险：
- 训练不稳定。
- 更难区分 reward 设计问题、观测接口问题和感知误差传播问题。

## 建议实验拆分

- `E1 Baseline Control`
  - 仅使用 `style reward + contact_count_reward`
  - 验证机器人能否稳定接触四类目标物
- `E2 Exploration Reward Ablation`
  - 比较是否加入 `novel_surface_grid_reward`
  - 比较是否加入 `geometry_weighted_novelty_reward`
  - 比较是否加入 `free_space_sweep_reward`
- `E3 Perception Model Sweep`
  - `Transformer` vs `RNN` vs `Diffusion-style decoder`
  - 对比体素重建质量、推理延迟、训练稳定性
- `E4 Joint Training Strategy`
  - `Option 1` 冻结底层 + 高层规划
  - `Option 2` 端到端联合训练

## 阶段性判断标准

- Must-have
  - 阶段 1：机器人能稳定接触圆柱体、立方体、墙面、墙角，并保持基本平衡与自然动作风格。
  - 阶段 2：感知模型能从纯本体感知历史重建出有意义的局部 3D 体素占据图。
  - 阶段 3：联合方案在“覆盖效率”或“建图质量”上优于阶段 1 + 阶段 2 的直接串联基线。
- Nice-to-have
  - 对未见过尺寸或摆放方式的障碍物保留一定泛化能力。
  - 学会主动优先探索边缘、角点等高信息密度区域。
- Stop condition
  - 若阶段 1 仍无法稳定产生可复用的手部探索动作，则暂停阶段 2 的大规模离线数据采集，先回到奖励与动作数据质量修正。
